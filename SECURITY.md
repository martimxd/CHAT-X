# Security Policy

SELF-HOST-SIGNAL-MESSENGER is an early self-hosted project. Treat it as a private deployment prototype until it has received independent security review.

Do not report sensitive vulnerability details in public issues. Open a private advisory or contact the maintainer through the repository owner profile.

This project does not claim Signal-grade security. It stores encrypted message payloads and encrypted media blobs, and it keeps cryptographic keys out of the server where the Phase 1 client supports that flow. Real Signal Protocol sessions are listed as Phase 2 work.
